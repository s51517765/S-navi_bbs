## メールアドレスおよびパスワード仕様
Djangoの標準（EmailField）におけるメールアドレスの有効判定は、基本的には「一般的で正しい形式か」をチェックしています。

具体的には、以下の3つのポイントがメインです。

1. 形式のチェック（バリデーション）
もっとも基本的なチェックです。以下の構成になっているかを見ています。

@（アットマーク） が1つだけ含まれているか。

@ の前（ユーザー名）と後ろ（ドメイン）が空ではないか。

ドメイン部分に .（ドット） が含まれているか（例: example.com）。

2. 文字種制限
基本的には半角英数字と一部の記号（., _, +, - など）のみ許可されます。

全角文字やスペースが入っているとエラーになります。

3. Djangoの「パスワードバリデーション」との兼ね合い
メールアドレス自体の条件ではありませんが、Djangoは標準で「ユーザー名やメールアドレスと似すぎているパスワード」を禁止しています。
もし登録ボタンを押して進まない場合、メールアドレスの中に含まれる文字列をパスワードに使おうとして、バリデーションエラーになっている可能性もあります。
